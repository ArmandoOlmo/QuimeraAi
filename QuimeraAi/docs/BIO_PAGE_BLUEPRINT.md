# Bio Page Blueprint

## Canonical Contract

`BioPageBlueprint` lives in `types/businessBlueprint.ts` as `businessBlueprint.bioPageBlueprint`. It is the AI Studio and BusinessBlueprint contract for generating a Bio Page draft without publishing it.

Core fields:

- `enabled`, `status`, `routeStrategy`, `defaultRoute`, `publicSlug`
- `title`, `description`, `profile`, `links`, `socialLinks`, `blocks`, `theme`
- `shop`, `booking`, `leadCapture`, `emailSubscribe`, `chatbot`, `analytics`, `seo`, `qrCode`
- `integrations`, `readiness`, `metadata`, `sourceMap`

`integrations` explicitly includes the ecosystem modules Bio Page Engine can coordinate: BusinessBlueprint, Design System, Ecommerce, Appointments, CRM, Email Marketing, ChatCore, Media AI, Analytics, and Website Builder.

## Draft Safety

Generated content must remain private until the user publishes from the Bio Page editor.

- Generated blocks and links should carry `generatedByAI = true` and `needsReview = true`.
- User changes should set `userModified = true`.
- Regeneration must preserve `userModified` and `lockedFromRegeneration` records.
- SEO drafts generated by AI should default to `noIndex = true`.
- ChatCore is represented as both a `chatbot` link and a `chatbot_cta` block so generated Bio Pages can behave like a link-in-bio funnel while still exposing an inline assistant CTA.
- AI must not invent follower counts, testimonials, products, booking availability, analytics, or reviews.
- Featured banner and contact blocks may be generated from known links/contact info, but they still remain review-required.
- Testimonials and FAQ blocks require explicit `WebsitePlan` source items. Empty, malformed, or incomplete proof/FAQ records are skipped instead of completed with invented content.

## Apply Flow

`services/bioPage/bioPageBlueprintService.ts` owns blueprint application:

- `previewBioPageBlueprintSync` returns a non-mutating summary before applying.
- `applyBioPageBlueprintDraft` creates or updates canonical draft data.
- `applyProjectBioPageBlueprintDraft` reads the blueprint from a project and applies it after AI Website Studio save.

The apply flow writes to `bio_pages`, `bio_page_blocks`, and `bio_page_links` through the Bio Page service layer.

Blueprint `sourceMap` is preserved when the canonical draft is created. Page-level source maps are stored in `bio_pages.settings.sourceMap`, block source maps are stored in `bio_page_blocks.settings.sourceMap`, and link source maps are stored in `bio_page_links.metadata.sourceMap`. This avoids a schema change while keeping enough provenance for review UI, regeneration locks, and debugging AI Studio output.
